Clock with fixed delay between frames (16ms), independent from any display/window.

It is used by withFrameNanos and withFrameMillis if one is not present in the calling kotlin.coroutines.CoroutineContext.

Use it only where you don't need to show animation in a window.

If you need a frame clock for changing the state of an animation that should be displayed to user, use MonotonicFrameClock that is bound to the current window. You can access it using LaunchedEffect:

LaunchedEffect {
  val frameClock = coroutineContext[MonotonicFrameClock]
}

Or using rememberCoroutineScope:

val scope = rememberCoroutineScope()
val frameClock = scope.coroutineContext[MonotonicFrameClock]

If withFrameNanos / withFrameMillis runs inside the coroutine scope obtained using LaunchedEffect or rememberCoroutineScope they also use MonotonicFrameClock which is bound to the current window.
